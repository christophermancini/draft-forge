# Repository Structure Manual

This document explains the technical structure of this DraftForge project.

## Directories

- **`chapters/`**: The source of truth for the manuscript. Files should be named sequentially (e.g., `01.md`, `02.md`) or with a prefix (e.g., `01-title.md`).
- **`docs/`**: World-building and research documentation.
    - `characters/`: Character sheets.
    - `world/`: Location and lore details.
- **`editorial/`**: Archive of past `EDITORIAL.md` files.
- **`stats/`**: JSON data generated by DraftForge (word counts, progress history).
- **`build/`**: Output directory for generated files (EPUB, PDF). This directory is git-ignored.
- **`.github/`**: GitHub Actions workflows for CI/CD.

## Configuration

- **`draftforge.yaml`**: Main configuration file.
- **`.gitignore`**: Standard git ignore rules.

## Automation

- **Stats Generation**: Runs on every push to `main`.
- **Ebook Build**: Runs on every push to `main`.
